4 类、对象和接口

4.1定义类继承结构

 kotlin接口和java不同：接口可以包含属性声明，默认声明是final和public的。另外，嵌套的类默认不是内部类

  4.1.1 Kotlin中的接口

  使用override修饰符是强制要求的。
  如果同样的继承成员有不止一个实现，必须提供一个显式实现。

  4.1.2 open、final和abstract修饰符，默认fnal

  脆弱的基类问题：对基类进行修改会导致子类不正确的行为。
  若想创建一个类的子类，需用open来修饰这个类，需被重写的属性和方法，也得加上open。
  抽象类和抽象成员始终是open，不用显式使用open。
  抽象类中的非抽象函数并不是默认open的，可以标记为open。

  4.1.3 可见性修饰符，默认为public

  java中的默认可见性——包私有，kotlin使用internal，表示“只在模块内部可见”，一个模块就是一组一起编译的kotlin文件。
  java中，同一个包中可以访问一个protected成员，在kotlin中，只有类和它的子类可见。

  4.1.4 内部类和嵌套类：默认是嵌套类

  嵌套类：部存储外部类的引用。
  嵌套类在java中，static class A，在kotlin中是classA。
  内部类：存储外部类的引用。
  内部类在java中，class A，在kotlin中，inner class A。

  4.1.5 密封类：定义受限的类继承机构

  密封类：用sealed修饰，这时默认为open类。
  用了密封类，when就不用再写else分支了。

4.3编译器生成的方法：数据类和类委托

  4.3.1 通用对象方法

  toString（）需要自定义。
  在java中，对于基本数据类型，==比较的值，对于引用类型，==比较的是引用。在kotlin中，==比较的是值，===比较引用。
  Hash容器：hashcode（）需要自定义实现。

  4.3.2 数据类：自动生成通用方法的实现

  类加载修饰符data，可以不用自定义实现toString、equals和hashcode
  为了让使用不可变对象的数据类变得更容易，用copy类的实例的方法，并在copy的同时修改某些属性的值。

  4.3.3 类委托：使用“by”关键字

  实现接口时，需要实现大量的代码，这时使用“by”，将接口的实现委托到另一个对象。

4.4“object”关键字：将声明一个类与创建一个实例结合起来

  4.4.1 对象声明：创建单例易如反掌

  object，对象声明将类声明与该类的单一实例声明结合在了一起。
  对象声明同样可以继承自类和接口，对象声明不允许有构造方法。

  4.4.2 伴生对象：工厂方法和静态成员的地盘

   修饰符为companion object

  4.4.3 作为普通对象使用的伴生对象

  声明一个命名伴生对象：companion object Loader
  伴生对象可以定义扩展函数。

  4.4.4 对象表达式：改变写法的匿名内部类

  java匿名内部类只能扩展一个类或实现一个接口，kotlin可以实现多个接口或不实现接口。
  与对象声明不同，匿名对象不是单例的，每次对象表达式被执行都会创建一个新的对象实例。
